# Views

stored sql select


> **references**
> - wikipedia: [sql view](https://en.wikipedia.org/wiki/View_(SQL))
> - w3schools: [sql view](https://www.w3schools.com/sql/sql_view.asp)

## create
```sql
CREATE VIEW _view_name_ AS  
SELECT _column1_, _column2_, ...  
FROM _table_name_  
WHERE _condition_;
```
## drop
with metadata 
```sql
IF EXISTS(SELECT * FROM sys.views WHERE name = '_view_name_'>) {
	DROP VIEW _view_name_
}
```

# CTE
> Common Table Expression

temporary result set in SQL that you can reference within a single query.

>**references**
>- geeks for geeks: [cte](https://www.geeksforgeeks.org/sql/cte-in-sql/)

## create
``` sql
WITH _cte_name_ AS (
SELECT * FROM _table_name_
) 
--must be used immediately
SELECT * FROM _cte_name_
```

## usage
-   Breaks complex queries into small, reusable parts.
-   Makes queries easier to read and understand.
-   Helps work with hierarchical data using recursion.

# Temporary tables
 Temporary tables are marked ``#_table_name`` and ``##_table_name``
 In comparison to CTE, tables are stored in a database.

 A table with a single ``#`` is stored locally, other connections cannot access it, and it is  freed after batch end(`` go``).

A table with ``##`` is stored globally, other connections can access it, and it is freed after no connection use it.

Tables are stored in **tempdb**.

> **references**
>-	geeks for geeks: [temporary tables](https://www.geeksforgeeks.org/sql/what-is-temporary-table-in-sql/)

## create
```sql
CREATE TABLE #_table_name (
	id int identity(1, 1) primary key not null,
	...
)
```
